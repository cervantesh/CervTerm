# CervTerm Release Trust Model

CervTerm beta releases are distributed as GitHub Release zip artifacts. This document explains what users can verify today and what is intentionally deferred.

## Current trust signals

For each tagged beta release, the release workflow publishes:

- Windows portable zip.
- Linux headless zip.
- `SHA256SUMS.txt`.
- GitHub build provenance attestations for the zip files and checksum file.

The release workflow runs from the repository tag and is visible in GitHub Actions.

## Verify checksums

Download all release assets into one directory, then run:

```bash
sha256sum -c SHA256SUMS.txt
```

On Windows PowerShell, verify an individual file with:

```powershell
Get-FileHash .\cervterm-<tag>-windows.zip -Algorithm SHA256
```

Compare the hash with the matching line in `SHA256SUMS.txt`.

## Verify provenance

GitHub artifact attestations are generated by the release workflow. Use GitHub's attestation tooling or the GitHub web UI to inspect provenance for the release assets.

Expected properties:

- The subject is a published release asset.
- The source repository is `cervantesh/CervTerm`.
- The workflow is `.github/workflows/release.yml`.
- The tag matches the release tag.

## Unsigned beta binaries

Current Windows beta zips are **not Authenticode signed**. This is intentional for the free beta path until a code-signing certificate and signing policy are available.

Implications:

- Windows SmartScreen may warn.
- Antivirus reputation may be limited.
- Users should verify checksums and provenance before running.

## Deferred trust improvements

Planned improvements:

1. Authenticode signing once certificate logistics are decided.
2. SBOM generation for release artifacts.
3. Dependency vulnerability scanning in CI.
4. Final winget package submission for the portable zip path.
5. MSI/WiX promotion only after install/update/config policy is settled.

## Maintainer release checklist

Before promoting a beta release:

1. CI is green on `main` and `dev`.
2. Local package preflight passes.
3. Installed-package smoke passes.
4. Release workflow succeeds.
5. Release is marked prerelease for beta tags.
6. `SHA256SUMS.txt` verifies downloaded assets.
7. Release notes mention unsigned status if signing is not enabled.
